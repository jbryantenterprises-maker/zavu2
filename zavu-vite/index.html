How Zavu works (the creative P2P twist)

1️⃣

You become the temporary server

When you create a link, your browser generates a P2P network ID and opens a WebRTC data channel. No file is uploaded anywhere.

2️⃣

Email the link (just like WeTransfer)

We open your email client with the link pre-filled. Recipient clicks → their browser connects directly to yours via WebRTC.

3️⃣

Pure peer-to-peer transfer

File is chunked and streamed live over encrypted WebRTC. Zero central storage. When you close the tab — link dies instantly.

4️⃣

Pro: Encrypted cloud links (7 days)

Pro users can optionally upload files encrypted to our cloud for 7-day persistent links. Files are encrypted client-side before upload — even we can't see them. Optional password protection adds another layer.

5️⃣

Creative extras

  • Real-time speedometer
  • Live progress on both sides
  • QR code for phone-to-phone magic
  • mailto: instant email composer
  • Zero-knowledge design — even we (the creators) can't see your files
